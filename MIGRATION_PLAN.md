# JCCI 平台迁移计划：Streamlit → Vue 3 + FastAPI

## 📋 项目概述

将现有的 Streamlit 单页应用迁移到 Vue 3 + FastAPI 前后端分离架构，提升交互体验、可维护性和扩展性。

### 核心原则

1. **保持后端核心逻辑不变**：JCCI 分析引擎、workflow1、调用链构建器等核心代码不做修改
2. **每步验证**：每个步骤完成后必须验证通过才能进入下一步
3. **无 TODO 遗留**：每一步都是完整可用的功能，不留未完成的工作给用户
4. **渐进式迁移**：先搭建基础架构，再逐步实现功能模块

---

## 🎯 阶段划分

### 阶段 1：后端 FastAPI 基础架构搭建（预计 3-5 天）

**目标**：建立 FastAPI 后端服务，复用现有业务逻辑，提供 RESTful API 和 WebSocket 支持

#### 任务清单

- [ ] **1.1 创建 FastAPI 项目目录结构**
  - 创建 `backend/` 目录
  - 建立标准 FastAPI 项目结构
  - 配置 `.gitignore` 和依赖文件
  
- [ ] **1.2 配置依赖和数据库连接**
  - 安装 FastAPI、uvicorn、pydantic、sqlalchemy
  - 配置 SQLite 数据库连接
  - 迁移 task_manager.db 表结构
  
- [ ] **1.3 实现任务管理 API（CRUD）**
  - POST `/api/tasks/submit` - 提交任务

  - GET `/api/tasks` - 获取任务列表
  - DELETE `/api/tasks/{task_id}` - 取消任务
  
- [ ] **1.4 集成现有 workflow1 和 task_manager**
  - 复制 `webapp/workflow1.py` 到 `backend/app/core/`
  - 复制 `webapp/task_manager.py` 到 `backend/app/services/`
  - 调整导入路径（不修改逻辑）
  
- [ ] **1.5 实现异步任务执行服务**
  - 使用 asyncio 或 Celery 实现后台任务
  - 集成埋点参数传递
  - 实现任务队列管理
  
- [ ] **1.6 添加 WebSocket 实时推送**
  - 实现 WebSocket 连接管理
  - 推送任务进度更新

- [ ] **1.7 验证：测试所有 API 端点**
  - 使用 Postman/curl 测试所有 API
