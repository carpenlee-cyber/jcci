# JCCI 部署说明

## 一、参数配置总览

所有可配置参数集中在 `backend/app/config.py`，支持三级优先级：

| 优先级 | 来源 | 说明 |
|--------|------|------|
| 1（最高） | 系统环境变量 | `export LLM_API_KEY=xxx` |
| 2 | `backend/.env` 文件 | 推荐生产环境使用 |
| 3（最低） | `config.py` 默认值 | 开发环境直接写死 |

> 复制 `backend/.env.example` → `backend/.env` 即可启用文件配置。

---

## 二、必填参数

### 2.1 LLM_API_URL

LLM API 服务端点地址。

| 项目 | 值 |
|------|-----|
| 变量名 | `LLM_API_URL` |
| 默认值 | `""`（空） |
| 示例 | `https://openai.good.hidns.vip/v1` |
| 说明 | 兼容 OpenAI 接口格式的任意 API 地址 |

### 2.2 LLM_API_KEY

LLM API 访问密钥。

| 项目 | 值 |
|------|-----|
| 变量名 | `LLM_API_KEY` |
| 默认值 | `""`（空） |
| 示例 | `sk-B882bCwUweSeMR...` |
| 安全提示 | **生产环境禁止写入 config.py**，使用 `.env` 文件并加入 `.gitignore` |

### 2.3 LLM_MODEL

使用的模型名称。

| 项目 | 值 |
|------|-----|
| 变量名 | `LLM_MODEL` |
| 默认值 | `""`（空） |
| 示例 | `moonshotai/kimi-k2.6`、`minimaxai/minimax-m2.7`、`gpt-3.5-turbo` |
| 说明 | 需与 API 服务商支持的模型名一致 |

> ⚠️ 三个参数任一为空时，AI 分析功能将自动降级为模拟数据。

## 三、服务器参数

### 3.1 HOST

后端服务绑定地址。

| 项目 | 值 |
|------|-----|
| 变量名 | `HOST` |
| 默认值 | `0.0.0.0` |
| 说明 | `0.0.0.0` 监听所有网络接口；仅本地访问可改为 `127.0.0.1` |

### 3.2 PORT

后端服务端口。

| 项目 | 值 |
|------|-----|
| 变量名 | `PORT` |
| 默认值 | `8000` |
| 说明 | 修改后需同步更新 `docker-compose.yml` 的端口映射和 `vite.config.ts` 的代理目标 |

---

## 四、数据库参数

### 4.1 DB_PATH

SQLite 数据库文件路径。

| 项目 | 值 |
|------|-----|
| 变量名 | `DB_PATH` |
| 默认值 | `backend/task_manager.db` |
| 说明 | 自动创建目录；Docker 中建议映射到 `/app/data/jcci.db` |

---

## 五、安全参数

### 5.1 DEBUG

调试模式开关。

| 项目 | 值 |
|------|-----|
| 变量名 | `DEBUG` |
| 默认值 | `False` |
| 说明 | 生产环境**必须**为 `False` |

### 5.2 ALLOWED_ORIGINS

CORS 允许的前端域名。

| 项目 | 值 |
|------|-----|
| 变量名 | `ALLOWED_ORIGINS` |
| 默认值 | `*`（允许所有来源） |
| 格式 | 逗号分隔的域名列表 |
| 生产示例 | `https://jcci.example.com,https://admin.example.com` |

## 六、前端参数

### 6.1 Vite 开发服务器端口

| 项目 | 值 |
|------|-----|
| 文件 | `frontend/vite.config.ts` |
| 参数 | `server.port` |
| 默认值 | `5173` |

### 6.2 Vite 开发代理

| 项目 | 值 |
|------|-----|
| 文件 | `frontend/vite.config.ts` |
| 参数 | `server.proxy./api.target` |
| 默认值 | `http://127.0.0.1:8000` |
| 说明 | 仅开发环境生效，将前端 `/api` 请求转发到后端 |

### 6.3 Nginx 代理（Docker 生产）

| 项目 | 值 |
|------|-----|
| 文件 | `frontend/nginx.conf` |
| 参数 | `proxy_pass` |
| 默认值 | `http://backend:8000` |
| 说明 | 使用 Docker 内部服务名，通常无需修改 |

## 七、Docker Compose 端口映射

| 服务 | 端口映射 | 说明 |
|------|---------|------|
| backend | `8000:8000` | 后端 API 端口 |
| frontend | `80:80` | 前端页面端口（80 被占可改为 `8080:80`） |

修改方式：

```yaml
# docker-compose.yml
services:
  frontend:
    ports:
      - "8080:80"   # 将主机 8080 映射到容器 80
```

## 八、快速部署清单

### 开发环境

端口8000由两个进程占用：PID 8204（服务端）和 PID 23772（客户端连接）。杀掉这两个进程：
netstat -ano | findstr :8000
taskkill /F /PID 8204; taskkill /F /PID 23772


```bash
# 1. 后端
cd backend; python -m uvicorn app.main:app --host 127.0.0.1 --port 8000 --reload --no-access-log

# 2. 前端（另一个终端）
cd frontend; npm run dev
# 访问 http://localhost:5173
```

### Docker 生产环境

```bash
# 1. 配置环境变量
cp backend/.env.example backend/.env
vi backend/.env   # 填入 LLM_API_URL、LLM_API_KEY、LLM_MODEL

# 2. 构建并启动
docker compose up -d --build

# 3. 查看状态
docker compose ps

# 访问 http://服务器IP:80
```
